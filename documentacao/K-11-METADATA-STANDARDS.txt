1. Introdução
Este documento define os atributos de metadados recomendados para cada classe de ativo do protocolo BBP.
O objetivo é fornecer orientação clara para criadores de ativos e templates, garantindo que aplicações possam interpretar corretamente os metadados mais comuns, sem perder a flexibilidade para atributos específicos de cada projeto.
________________________________________
2. Princípios Gerais
Princípio	Descrição
Recomendação, não imposição	Criadores podem incluir atributos adicionais livremente
Hierarquia de classes	Subclasses herdam atributos da classe pai
Namespaces para extensões	Atributos específicos devem usar namespace (ex: game.attack_power)
Tipos definidos	Cada atributo tem um tipo esperado (string, number, boolean, array, object)
Atributos estatísticos	Para coleções com raridade, usar trait com percentage

3. Classe: ART
Ativos artísticos digitais ou físicos.
DNA: BBP:id:art:namespace:identifier
Standard no manifesto: nft, generative_art, physical_art
Atributos Recomendados
Atributo	Tipo	Descrição	Obrigatório
title	string	Título da obra	Recomendado
creator	string	Criador da obra	Recomendado
year	number	Ano de criação	Opcional
medium	string	Meio (óleo, digital, fotografia, etc)	Opcional
collection	string	Coleção à qual pertence	Opcional
edition	string	Edição (ex: "1/100", "open")	Se aplicável
license	string	Licença de uso	Recomendado
description	string	Descrição da obra	Opcional
image	string	Hash ou URI da imagem principal	Recomendado
Atributos para Coleções com Raridade (Trait-based)
Para NFTs compostos por camadas com porcentagens:
Atributo	Tipo	Descrição	Exemplo
traits	array	Lista de camadas/características	Ver abaixo
Formato recomendado para traits:
json
"traits": [
  {
    "trait_type": "background",
    "value": "sunset",
    "percentage": 5.2
  },
  {
    "trait_type": "body",
    "value": "golden",
    "percentage": 1.8
  },
  {
    "trait_type": "eyes",
    "value": "laser",
    "percentage": 3.1
  },
  {
    "trait_type": "accessory",
    "value": "crown",
    "percentage": 0.5
  }
]
O campo percentage indica a raridade daquela característica na coleção (quanto menor, mais raro).
Exemplo Completo
json
"attributes": {
  "title": "Fredo Genesis #2654",
  "creator": "Leo",
  "collection": "fredo_genesis",
  "year": 2026,
  "medium": "digital",
  "license": "CC-BY-NC 4.0",
  "traits": [
    {
      "trait_type": "background",
      "value": "cosmic",
      "percentage": 2.3
    },
    {
      "trait_type": "body",
      "value": "legendary_gold",
      "percentage": 0.7
    },
    {
      "trait_type": "eyes",
      "value": "laser",
      "percentage": 3.1
    }
  ]
}

5. Classe: GAME_ASSET
Itens utilizados em jogos ou mundos virtuais.
DNA: BBP:id:game_asset:namespace:identifier
Standard no manifesto: game_item, character, skin, weapon
Atributos Recomendados
Atributo	Tipo	Descrição	Obrigatório
game	string	Jogo ao qual pertence	Recomendado
asset_type	string	Tipo (weapon, skin, character, land, etc)	Recomendado
name	string	Nome do item	Recomendado
rarity	string	Raridade	Recomendado
level	number	Nível do item	Opcional
is_tradable	boolean	Se pode ser negociado	Recomendado
is_soulbound	boolean	Se é vinculado ao personagem	Opcional
Atributos Específicos (com namespace)
Atributo	Tipo	Descrição
game.attack_power	number	Poder de ataque
game.defense	number	Defesa
game.durability	number	Durabilidade
game.required_level	number	Nível necessário
game.slot	string	Slot do inventário
game.effects	array	Efeitos especiais
Exemplo
json
"attributes": {
  "game": "medieval_rpg",
  "asset_type": "weapon",
  "name": "Dragonslayer Sword",
  "rarity": "legendary",
  "level": 50,
  "is_tradable": true,
  "game.attack_power": 250,
  "game.durability": 100,
  "game.required_level": 45,
  "game.effects": ["fire_damage", "life_steal"]
}

7. Classe: COMMODITY
Commodities e produtos do mundo real.
DNA: BBP:id:commodity:namespace:identifier
Standard no manifesto: agricultural, mineral, energy, precious_metal
Atributos Recomendados
Atributo	Tipo	Descrição	Obrigatório
commodity_type	string	Tipo (coffee, gold, oil, wheat, etc)	Recomendado
origin	string	Origem geográfica	Recomendado
harvest_year	number	Ano da safra/extração	Se aplicável
grade	string	Classificação (ex: specialty, arabica)	Recomendado
purity	number	Pureza (para metais, 0.999 = 99.9%)	Se aplicável
quantity	number	Quantidade	Recomendado
unit	string	Unidade (kg, oz, bag, barrel)	Recomendado
certifications	array	Certificações (fair trade, organic, etc)	Opcional
warehouse	string	Localização do armazém	Se aplicável
lot_number	string	Número do lote	Recomendado
Atributos Específicos por Tipo
Café:
Atributo	Tipo	Descrição
coffee.species	string	Arábica, Robusta, etc
coffee.variety	string	Bourbon, Catuai, etc
coffee.process	string	Natural, washed, honey
coffee.altitude	number	Altitude em metros
coffee.cupping_score	number	Pontuação (0-100)
Ouro/Prata:
Atributo	Tipo	Descrição
metal.purity	number	Pureza em porcentagem
metal.weight	number	Peso
metal.form	string	Barra, moeda, pó
metal.refinery	string	Refinaria certificada
Petróleo:
Atributo	Tipo	Descrição
oil.api_gravity	number	Densidade API
oil.sulfur_content	number	Teor de enxofre
oil.benchmark	string	Brent, WTI, etc

  "lot_number": "MG-2026-001",
  "coffee.species": "arabica",
  "coffee.variety": "yellow_bourbon",
  "coffee.process": "natural",
  "coffee.altitude": 1150,
  "coffee.cupping_score": 86
}
Exemplo (Ouro)

"attributes": {
  "commodity_type": "gold",
  "origin": "South Africa",
  "purity": 0.9999,
  "quantity": 100,
  "unit": "troy_ounce",

20. Como Usar Este Documento
Criadores de ativos devem:
1.	Identificar a classe do ativo conforme BBP-0006
2.	Consultar a seção correspondente neste documento
3.	Incluir os atributos recomendados no manifesto
4.	Adicionar atributos específicos com namespace quando necessário
5.	Usar o formato traits com percentage para coleções com raridade
Indexadores e resolvers podem:
•	Reconhecer atributos padrão automaticamente
•	Extrair metadados consistentes para exibição
•	Calcular raridades baseado nos percentage dos traits
•	Armazenar atributos com namespace para consultas especializadas
