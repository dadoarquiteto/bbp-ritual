BBP-0004 — Resolver & Indexing Architecture
Status
Draft
Título
Arquitetura do Resolver BBP
Resumo
O Resolver BBP é o componente responsável por transformar identificadores BBP em informações utilizáveis sobre um ativo digital.
Ele realiza esse processo recuperando o manifesto do ativo, consultando dados em redes blockchain através de indexadores públicos ou provedores RPC, e retornando um resultado padronizado de resolução.
Resolvers podem existir em diferentes ambientes, incluindo aplicações, navegadores ou nós de rede dedicados. O protocolo não depende de infraestrutura centralizada; múltiplos resolvers independentes podem coexistir.
O papel do resolver é exclusivamente ler e agregar estados de redes externas, não realizar consenso nem modificar estados de blockchain.
________________________________________
1. Visão Geral
O processo de resolução BBP transforma um identificador BBP em uma representação completa de um ativo digital.
Exemplo de identificador:
BBP:id:art:fredo:2654
O processo de resolução ocorre nas seguintes etapas:
BBP Identifier
↓
Registry lookup
↓
Manifest retrieval
↓
Blockchain state queries
↓
Resolution result
O resolver atua como a camada que orquestra essas operações.
________________________________________
2. Responsabilidades do Resolver
Um resolver executa as seguintes funções principais:
1.	Interpretar o identificador BBP
2.	Consultar o Registry BBP
3.	Recuperar o Manifesto do ativo
4.	Interpretar os módulos definidos no manifesto
5.	Consultar dados em redes blockchain
6.	Produzir um resultado padronizado de resolução
Resolvers não armazenam estado autoritativo.
Eles apenas recuperam e agregam dados provenientes de redes externas.
________________________________________
3. Modelos de Implementação
Resolvers podem existir em diferentes contextos de execução.
3.1 Resolver Embutido em Aplicações
Resolvers podem funcionar diretamente dentro de aplicações, como:
•	visualizadores de ativos
•	carteiras digitais
•	exploradores de blockchain
•	navegadores especializados
Arquitetura típica:
User
↓
Viewer Application
↓
Embedded Resolver
↓
Registry
↓
Manifest
↓
Indexers
Esse modelo permite implementações leves e não requer infraestrutura própria.
O viewer desenvolvido para o protocolo BBP é um exemplo de aplicação que pode incluir um resolver embutido.
________________________________________
3.2 Resolver como Nó de Rede
Resolvers também podem operar como serviços independentes dentro de uma rede descentralizada.
Exemplo de arquitetura:
Client
↓
Resolver Node
↓
Indexers
↓
Blockchains
Esses nós podem oferecer APIs públicas que permitem que aplicações consultem ativos BBP.
O comportamento da rede de nós resolver é definido no documento:
BBP-0007 — Resolver Nodes
________________________________________
4. Interpretação do Manifesto
O resolver interpreta os módulos definidos na especificação de Manifesto BBP.
Módulos comuns incluem:
dna
chains
assets
events
attributes
render
Exemplo de definição de eventos no manifesto:
events:
  - Transfer
  - Mint
Essas declarações indicam quais eventos ou estados devem ser observados nas redes blockchain.
O resolver traduz essas declarações em consultas específicas para cada rede.
________________________________________
5. Fontes de Dados
Resolvers obtêm informações de redes blockchain utilizando infraestrutura pública disponível, como:
•	indexadores públicos
•	endpoints RPC
•	gateways de rede
Exemplos incluem serviços equivalentes a:
•	Mempool.space
•	Infura
•	Alchemy
Resolvers podem consultar múltiplas fontes para aumentar a confiabilidade das respostas.
Os detalhes de integração com cada rede são definidos em:
BBP-0008 — Network Adapters
________________________________________
6. Arquitetura Stateless
Resolvers são projetados para serem stateless.
Isso significa que eles não mantêm estado global nem participam de mecanismos de consenso.
Sua função é limitada a:
retrieve
interpret
aggregate
return
Como consequência, múltiplos resolvers independentes podem operar simultaneamente sem necessidade de coordenação.
________________________________________
7. Modelo de Confiança
Resolvers dependem de infraestrutura pública para recuperar dados de redes blockchain.
Implementações podem opcionalmente incluir mecanismos como:
•	múltiplos indexadores para redundância
•	fallback automático entre provedores
•	verificação de respostas
•	cache temporário
Essas estratégias aumentam a confiabilidade, mas não são exigidas pelo protocolo.
________________________________________
8. Resultado da Resolução
Resolvers retornam um resultado estruturado definido na especificação:
BBP-0005 — Resolution Format
O resultado normalmente inclui informações como:
asset metadata
ownership
network states
render data
Aplicações utilizam esses dados para apresentar o ativo ao usuário.
________________________________________
9. Papel na Arquitetura BBP
O resolver conecta os principais componentes do protocolo:
DNA
↓
Registry
↓
Manifest
↓
Resolver
↓
Network Adapters
↓
Blockchains
Essa camada torna possível transformar identificadores BBP em ativos utilizáveis em aplicações.
________________________________________
10. Conclusão
O Resolver BBP é um componente flexível responsável pela resolução de ativos no protocolo.
Principais características:
•	descentralizado
•	stateless
•	leitura de estado
•	independente de ambiente
Resolvers podem operar em navegadores, aplicações ou nós de rede, permitindo um ecossistema distribuído de serviços de resolução.
