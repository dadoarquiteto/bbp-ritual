BBP-0003 – Registry Standard

Esse documento resolve um problema enorme em protocolos:
caos semântico.

Sem um registry, cada projeto escreve coisas diferentes.

Exemplo ruim:
creator
artist
author
maker

Tudo significa a mesma coisa.
Indexadores ficam perdidos.

________________________________________

**BBP-0003 — Registry Standard**

Define o **vocabulário oficial do protocolo BBP**, alinhado com as classes definidas no **BBP-0006 (Universal Asset Classes)**.

Ele padroniza:

- tipos de ativos
- atributos
- relações
- eventos

---

## 1. Estrutura do Registry

O registry contém quatro categorias principais:

- types
- attributes
- relations
- events

---

## 2. Types Registry

Define os tipos de ativos permitidos, conforme a taxonomia oficial do **BBP-0006**.

Exemplos:

| Type           | Classe BBP-0006 | Descrição |
|----------------|------------------|-----------|
| art            | ART              | Ativos artísticos |
| collectible    | COLLECTIBLE      | Itens colecionáveis |
| token          | TOKEN            | Ativos fungíveis |
| game_asset     | GAME_ASSET       | Itens para jogos |
| identity       | IDENTITY         | Identidades digitais |
| domain         | DOMAIN           | Nomes descentralizados |
| dataset        | DATASET          | Conjuntos de dados |
| ai_model       | AI_MODEL         | Modelos de IA |
| rwa            | RWA              | Ativos do mundo real |
| ticket         | TICKET           | Ingressos digitais |
| license        | LICENSE          | Licenças digitais |
| membership     | MEMBERSHIP       | Associações e acessos |
| **ritual_seed**| **RITUAL_SEED**  | **Participante do ritual BBP (NOVO)** |

Exemplos de DNAs válidos:
BBP:id:art:fredo:2654
BBP:id:domain:leo
BBP:id:game_asset:sword:001
BBP:id:token:usdc:main
BBP:id:rwa:property:001
**BBP:id:ritual_seed:scene_1:bc1p...xyz (NOVO)**

Isso **não muda o DNA**, apenas padroniza interpretação.

---

## 3. Attributes Registry

Define atributos padronizados para manifestos.

Exemplos:

| Attribute  | Tipo     | Descrição |
|------------|----------|-----------|
| creator    | string   | Criador do ativo |
| collection | string   | Coleção à qual pertence |
| rarity     | string   | Raridade (common, rare, legendary) |
| edition    | number   | Número da edição |
| license    | string   | Tipo de licença |
| description| string   | Descrição do ativo |
| image      | string   | Hash ou URI da imagem |
| **bbp_network_version** | **string** | **Versão da rede BBP (ex: "2.0") (NOVO)** |
| **distribution_method** | **string** | **Método de distribuição ("automatic" / "manual") (NOVO)** |

Exemplo dentro do manifesto:

```json
"attributes": {
  "creator": "leo",
  "collection": "fredo_genesis",
  "rarity": "legendary",
  "bbp_network_version": "2.0",
  "distribution_method": "automatic"
}
4. Relations Registry
Define relações entre ativos.

Exemplos:

Relation	Descrição
derived_from	Ativo derivado de outro
contains	Ativo contém outro
references	Ativo referencia outro
tokenizes	Ativo tokeniza outro
controls	Ativo controla outro
represents	Ativo representa outro
registered_on	Ativo registrado em uma rede específica (NOVO)
Exemplo:
Fredo NFT derived_from Fredo Collection
Seed registered_on BBPNetwork (NOVO)

5. Events Registry
Define eventos observáveis nas redes.

Exemplos:

Event	Descrição
mint	Ativo foi criado
transfer	Ativo foi transferido
burn	Ativo foi destruído
sale	Ativo foi vendido
bridge	Ativo foi bridgeado para outra rede
stake	Ativo foi staked
unstake	Ativo foi unstaked
seed_activated	Participante conectou carteira e se tornou seed ativa (NOVO)
fraction_distributed	Frações do protocolo foram distribuídas (NOVO)
Esses eventos são usados no módulo events do manifesto.

6. Estrutura de entrada do Registry
Cada item do registry segue formato simples.

Exemplo:

json
{
  "id": "creator",
  "category": "attribute",
  "type": "string",
  "description": "Creator of the asset"
}
NOVO - Exemplo para seed_activated:

json
{
  "id": "seed_activated",
  "category": "event",
  "type": "object",
  "description": "Participant connected wallet and became an active seed",
  "properties": {
    "wallet_address": "string",
    "scene_id": "number",
    "timestamp": "number"
  }
}
7. Registry extensível
Projetos podem criar extensões usando namespaces.

Exemplos:

game.attack_power

game.defense

ai.model_type

ai.training_dataset

rwa.location

rwa.legal_document

bbp.network_version (NOVO)

bbp.distribution_method (NOVO)

Isso mantém o protocolo flexível e escalável.

8. Onde o Registry é usado
O registry influencia:

manifestos

indexadores

resolvers

marketplaces

templates

BBPNetwork (NOVO)

Ele garante que todos interpretem os dados da mesma forma.
